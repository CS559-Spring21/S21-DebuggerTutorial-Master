Tracking Values

The main use of console.log is to check on certain values or write messages that indicate if the code has gone through a certain point. We can do that much more easily using the Log Message option when editing a breakpoint. This will create a logpoint.

Using the logpoint, we can type in a message at any point in the code. NOTE: we cannot set a breakpoint on top of a logpoint. If there is an expression or hit count set, it will override the logpoint and no message will appear.

With the logpoint you can also print values as you would using console.log by adding {} around your variables.

Box (Exercise): Logging

Below is the same program as before but it is in 4-1.js. Let us add the following logpoint to line 23:

red: {r}, green: {g}, blue: {b}

When we run this, it will log the values of r, g, and b. You can read this by looking at the debug console on the lower tab of VS Code, or by pressing Ctrl + Shift + i on the page and look at the console there. If we made this logpoint somewhere inside the loop, say on line 27, it will log the rgb values on each iteration.

Watching Values

In the debug tab, there are three sections: Variables, Watch, and Call Stack. We already talked about the call stack, but not the other two.

Variables contains all the variables currently saved while running your program. If we set a breakpoint on line 24 of 4-1.js and run the debugger, we can find the variables the rgb values. You can also directly modify values of variables while debugging at a breakpoint.

Watch holds the variables we wish to constantly track. If we right click on a variable in Variables, we can Add to Watch. Try adding the 3 rgb values to Watch. If we keep continuing at at that breakpoint on line 24, we can see the values shift on each iteration.

Summary

Essentially you can still log messages and values in the console when you need. Hopefully doing it this way will prove to be faster also leave less of a mess behind when you no longer need the log messages. Also watching values should prove to be useful since you don't have to constantly log values at various points in time during your code.
