Getting Started

I will start by saying that you don't have to use the debugger in VS Code, since Google Chrome's devtools allow you to debug directly in the browser. If you press Ctrl + Shift + i and go to Sources, you can look for the js file you want to debug and directly add breakpoints to your code there.

For the exercise box at the bottom of this page, it would be:

1-1.html -> 127.0.0.1:[port] -> for_students -> 1-1.js

If you are fine with using the debugger in Chrome, you can skip to the exercise at the bottom and go from there. But in my opinion, the debugger in VS Code looks much nicer.

Setting up

For this setup, we're going to be using Chrome but this also works with Firefox as well as other browsers.

First, install the debugger for Chrome. You can find it by going to the extensions tab found on the left of the VS Code window. Search up 'Debugger for Chrome' and install it.

Run the live server for your page (It should already be running for this Debugger WB).

The next step depends on whether you are working on a single folder or are working in a workspace. (Generally speaking, a workspace will have multiple folders)

If working in a single folder

Click on the run/debug tab to the left. Click on Run and Debug.

If working in a workspace

Click on the run/debug tab to the left. Go to the top and click the scrolldown. Click on 'Add Config (Debugging WB)'.

The steps after should be the same

It will then ask you to select an environment, in which you will just select what browser you use to open your page, which for this example will be Chrome. If you didn't install the Debugger for Chrome, Chrome will not show up as an option.

Now this launch.json file should open up and it can be found in the newly added .vscode folder of this workbook. We need to modify the url so that the port matches the one we are using for the live server.

We are now done with setting up the debugger for VS Code.

Note: we only have to do this once per folder. If you open up a folder that you already set up the debugger for, the launch settings will be saved in launch.json in the .vscode folder.

Running the debugger

Now try running the debugger. It can only run when the live server is open. You can run the debugger by click the arrow next to your launch tab.

It should open a new window with your page site. In VS Code, you should see this debugger bar at the top of VS Code. You'll also notice that the bar at the bottom turns orange.

Box (Exercise): First breakpoints

Now lets set some breakpoints. Below is a canvas with four squares. The javascript for this can be found in 1-1.js. You will see that there are four functions and each gets called to draw a square. To the left of the line number for any line, you can click to add a breakpoint, or click again to remove it.

Lets add a breakpoint on each function (lines 27-30).

In the new window, go to this page. It should be paused in the debugger bar in VS Code. If you set the breakpoints correctly, you will see that the canvas is empty. You can click on continue on the debugger bar to go to the next breakpoint. You will notice that each square will appear as you continue as the code is paused at each breakpoint.

One more thing of note, make sure you continue all the way or remove/disable all breakpoints and refresh when you want to leave the page while debugging. You won't be able to leave the page until you do. Of course, if you are done debugging, you can simply end the debugging session anytime by closing out the window.

At the bottom of the debug tab, there is a Breakpoints tab. If you hover over the tab, you will see 3 actions you can do, the one on the far right will allow you to instantly remove all breakpoints.

Summary

This was essentally how we setup the VS Code debugger to run. You can of course just use the Chrome devtools if you prefer those, but going forward, it shouldn't matter which you use. Next we will look at stepping through our code.
