Stepping

From the last page we saw how to continue to the next breakpoint, but there are some other things we to progress through our code in the debugger bar.

From the bar you can do the following things:

  • Continue/Pause: Continues to the next breakpoint or end of program. Pauses the program if it is continuously running in say an animation loop.
  • Step Over: Goes to the immediate next line of code.
  • Step In: Goes into the next line in a function if applicable, otherwise goes to the immediate next line of code
  • Step Out: Goes out of the function it is currently in and continues to next line
  • Restart: Refreshes the page
  • Stop: Ends debugging and closes the window

If we step over, we will get a pink square and if we step over against, we will get the green square. You will notice that the debugger simply goes to the next line of code regardless what happens in the function.

Lets restart step in this time. Now you will see that the debugger goes into func1(), and if you keep stepping in, we will go into func3(), complete it, go back into func1(), complete it, and then the same for func2(). You will see that we drew a blue square, and pink square, and a green square in the order.

Lets restart and step into func3() again. This time step out as soon as we get into func3(). You will see that it immediately completes the function and goes back into func1() and draw the blue square. If we were to step out after immediately entering func1(), we will not see the blue square because the pink square is also immedately drawn after. Essentially, step out finishes the current function and continues to the next line after the function call.

The Call Stack

You can tell where we will continue from by looking at the Call Stack in the debugger tab. The image below shows what the Call Stack should look like when we step into func3(). We can see that func3() is at the top of the stack, and we can see that if we step out, we will back into func1(), and then if we step out again we would be in the window.onload() function.

Summary

These are the basics of how to step through your programs. Feel free to play around with this some more if you are not entirely sure how this works.
